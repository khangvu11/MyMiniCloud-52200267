3 tips nhớ lâu khi làm lab Docker

  1. Vẽ sơ đồ trước khi viết compose: hiểu rõ service nào nói chuyện với ai, port nào dùng cho nội bộ, port nào publish ra ngoài.
  2. Đặt tên container rõ ràng: ví dụ web-frontend-server, application-backend-server giúp đọc log và ping trong mạng dễ hơn.
  3. Luôn kiểm tra bằng curl hoặc ping: sau khi chỉnh cấu hình, dùng curl / dig / ping để chắc chắn service hoạt động đúng.
